== 信用手形で掛け売りをする場合

貸方、または、借方の機能は、（例えば、380 や 381 といった）ビジネス文書の種類によってのみ制御されます。種類によって合計金額の符号は影響されません。

<1> 381 は信用手形を表す
<2> 手数料
<3> 正の数量と単価の請求明細1行目
<4> 負の数量と単価の請求明細2行目

== ネガティブ請求書で請求する場合

貸方、または、借方の機能は、請求金額の符号（つまり、正、または、負）によってのみ制御されますが、ビジネス文書の種類（例えば、380）は、操作（「貸方」）自体とは関係ありません。

.上の例の請求書を訂正するネガティブ請求書の UBL での例
[source, xml, indent=0]
----
include::{snippet-dir}/ja/base-negative-inv-correction.xml[tags=negative]
----
<1> 380 は請求書を表す
<2> 元の請求書を修正するための負の小計
<3> 全ての文書レベルでの合計は負になっています
<4> 元の請求書では正の数量と単価であった請求明細1行目は両方とも負となっている
<5> 単価は常に正である必要があり、変更はなし
<6> 元の請求書では負の数量と単価であった請求明細2行目は両方とも正となっている
