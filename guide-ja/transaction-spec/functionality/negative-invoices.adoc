
= ネガティブ請求書と信用手形取引


本 BIS では負の総計をサポートします。

ネガティブ請求書を採用する理由は、請求書発行プロセスの幅を広げるためです。

そのような処理の例は以下の通りです。

*	後に請求される従量課金での請求書で調節されるが取られる事前（推定）消費請求書。
*	Pre-payment (with or without CT) is settled through a final invoice; and
* 一部の人は、帳簿を修正する場合に、信用手形ではなくマイナスの請求書を使用することを好みます。

====
NOTE: 電子請求書と注文または請求対象との自動照合を重視する買い手は、複雑なトランザクションを受け入れることができる領域と状況を制限し、調達時に要件を明示することをお勧めします。
====

この決定は、取引の形式に次の影響を及ぼします:

* 請求書（「ネガティブ請求書」）は、信用手形の代替として機能できます。請求書生成システムはどちらを利用しても実装できますが、請求書受信システムは両方をサポートする必要があります。
*	信用手形の取引形式は、負の総計にも対応できるように設計する必要があります。これは、マイナスの請求書全体を信用手形で相殺する必要がある場合があるためです。

与信情報の伝達に関して、信用手形とネガティブ請求書の本質的な違いがあることに注意する必要があります。
